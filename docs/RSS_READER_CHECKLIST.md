# RSS 阅读器待完善清单

更新时间：2026-03-29

## 文档目的

这份文档专门记录 RSS “阅读器层”的当前状态、待完善项和目标效果。

这里的“阅读器”不是新的抓取器或新的 AI 能力，而是 RSS 内容进入详情页之后，如何被更稳定、更舒服地展示出来。

它主要覆盖：

- 阅读模式切换
- 悬浮目录与跳转
- 正文排版与层级
- 图片与附件的基础展示
- 空态、异常态与降级提示

当前阶段的原则是：

- 以“完善现有 RSS 阅读链路”为主
- 不再额外扩展新的大功能方向
- 优先做文字 / 文章型 RSS 的阅读体验收尾
- 图片相关只保留“不影响阅读”的基础能力

## 当前收尾方向

本轮阅读器优化已经明确聚焦“文字 / 文章型 RSS 源”，优先处理：

- 正文排版清晰度
- 小标题目录跳转
- 阅读模式切换简化
- 详情页布局减负

同时已明确：

- 当前不再推进“长文默认折叠”方案
- 阅读模式仍然只有 `原始 / 增强 / 摘要` 三种
- 目录会继续走“轻盈、悬浮、不抢正文注意力”的方向

## 当前已具备的阅读器基础

基于当前仓库代码，RSS 阅读器已经具备以下核心能力。

### 1. 阅读模式

- 已有 `原始 / 增强 / 摘要` 三种阅读模式。
- 阅读模式按钮已并入标题下方的灰色元信息行右侧。
- 三个模式已改为图标按钮，不再额外展示“阅读模式”文字标题。
- 按钮已有独立悬浮提示、激活态和禁用态。
- 打开 RSS 文章时会根据内容状态自动选择默认模式：
  - 有独立摘要时优先落到“摘要”
  - 没有独立摘要时默认落到“增强”
- 当某种模式没有实际差异时，界面会显示解释性提示，而不是静默失败。

### 2. 悬浮目录

- 已有左侧悬浮目录。
- 目录默认以极细的“竹节杆”形态收起，悬浮后向右展开。
- 目录会根据正文实际标题自动生成。
- 如果渲染后的 HTML 标题不足，也会回退到 `content_blocks` 或 Markdown 标题中提取目录。
- 点击目录项时可平滑滚动到对应位置。
- 如果当前处于“摘要”模式且目标标题只存在于正文模式，目录跳转会自动切回 `增强` 或 `原始` 后再定位。
- 目录当前已支持基础的“当前章节高亮”。

### 3. 正文展示

- 已支持 Markdown 渲染。
- 已支持标题、段落、列表、引用、图片、链接等常见结构的展示。
- 正文阅读当前以“完整展开”作为默认策略，不再走长文折叠方案。
- 详情页中原来的 RSS 结构摘要标签已移除，减少视觉噪音。
- 正文区域已形成“标题 / 元信息 / 正文主体”的基础阅读层级。

### 4. 图片与附件基础

- 已支持封面图 Hero 区展示。
- 已支持按图片类别展示图集：
  - 正文图
  - 附件图
  - 外链图
- 已支持正文内嵌图片点击预览。
- 已支持图片加载失败时的前端占位和降级提示。
- RSS 附件区已和阅读模式联动，避免摘要模式下信息过载。

说明：

- 这部分目前保留“基础可用”即可，不是当前主优化重点。

### 5. 状态联动

- 切换文章时会重置目录状态、图片预览和图片错误状态。
- 切换阅读模式时会重新刷新正文和目录匹配关系。
- RSS 规则预览与详情阅读模式已经能基本闭环，不再是割裂的两套结果。
- 后台静默刷新列表时，不会再悄悄改动用户当前正在看的 RSS 阅读模式。

如果只看“能不能用”，当前阅读器已经可用。

如果看“像不像一个成熟的 RSS 阅读界面”，还差最后一轮系统性的收尾。

## 待完善项

下面按优先级从高到低整理。

## P0：必须收口

这些项不属于“锦上添花”，而是决定阅读器是否算真正收尾。

### 1. 悬浮目录细节收边

当前目录主形态已经落地，但还可以继续打磨：

- 当前阅读位置的高亮反馈还能更稳定一些
- 目录展开 / 收起的节奏还能更丝滑
- 标题很多时，目录自身的滚动与可见范围还可以更清楚
- 小窗口下目录与正文的距离还需要持续压测
- 短文或标题很少时，目录是否需要进一步弱化，还可以再判断

目标效果：

- 平时几乎不打扰阅读
- 需要时又能很快跳到目标位置
- 长文场景下像一个轻量阅读导航器，而不是侧栏面板

### 2. 三种阅读模式的结果边界再清晰一点

当前模式切换已经有了，但用户感知上还可以更清楚：

- 原始模式：明确是“解析后的原始 Markdown”
- 增强模式：明确是“排版增强后的正文”
- 摘要模式：明确是“压缩内容 + AI 标签”

建议完善：

- 模式提示语再统一
- 模式切换后的差异感再直观一点
- 对“增强和原始一致”的情况，用更轻但更明确的提示表达

目标效果：

- 用户一眼知道自己正在看哪种内容
- 不会怀疑“是不是没切成功”

### 3. 文字正文排版继续收边

当前 Markdown 基础样式已经有，但文字型 RSS 仍然值得继续打磨：

- 标题层级的节奏
- 段落间距和首屏密度
- 引用块的层次感
- 列表缩进和列表项间距
- 长链接断行后的可读性

目标效果：

- 不同来源的 RSS 正文看起来更整齐
- 不是“能显示”，而是“看着顺、读着稳”

### 4. 窄窗口与默认 GUI 尺寸继续压测

当前已经做过一轮适配，但仍值得继续压：

- 元信息行与阅读模式按钮的挤压关系
- 目录和正文之间的呼吸空间
- 长标题换行后的节奏
- 小尺寸窗口下正文左右留白是否仍然舒服

目标效果：

- 在默认小窗口下也不显得拥挤
- 不会因为目录和按钮加入后破坏原本的阅读节奏

### 5. 空态 / 异常态统一

建议统一几类场景：

- 无正文
- 正文过短
- 没有摘要
- 增强内容与原始内容一致
- 图片加载失败

目标效果：

- 每种异常都能给用户一个清楚、轻量、不打扰的解释

## P1：重点打磨

这些项不一定阻塞上线，但会明显影响“像不像产品”。

### 1. 目录增加更强的阅读进度感

- 当前章节高亮可以更明显
- 可考虑给目录加入“阅读进度”暗示
- 长文阅读时可让目录对当前所在区域的反馈更连续

### 2. 模式记忆策略

- 可考虑记住用户最近一次偏好的 RSS 阅读模式
- 或按文章来源类型记忆默认模式

说明：

- 这属于体验优化，不是必须项。
- 当前已有“按文章自动选默认模式”的基础逻辑。

### 3. 标题、摘要、正文第一屏的视觉节奏再统一

- 标题过长时的断行节奏
- 标签、摘要、正文第一屏的密度
- 纯文字长文首屏的信息聚焦

### 4. 复制 / 截图 / 分享结果与当前模式更强一致

当前复制链路已会根据阅读模式选择正文，但仍可继续统一：

- 摘要模式复制出来更像“简报”
- 增强模式截图更像“阅读卡片”
- 原始模式导出更保留原始结构

## P2：收尾增强

这些项不急，且目前应继续后置。

### 1. 图集区的“查看更多”闭环

- 超过首屏数量时，补齐“查看更多图片”
- 让用户知道还有多少张未展开
- 展开后保持阅读位置稳定

### 2. 图片预览弹层继续打磨

- 增加上一张 / 下一张切换
- 增加键盘关闭或翻图支持
- 对超长图、超大图的缩放体验更友好

### 3. 正文内嵌图片与图集区的关系统一

- 减少重复图片带来的割裂感
- 统一外链图、附件图、正文图的说明文案
- 统一图片点击反馈与命名方式

## 推荐收尾顺序

如果只做“完善，不加新方向”，建议按下面顺序推进：

1. 悬浮目录细节收边
2. 三种阅读模式的文案和差异提示统一
3. 文字正文排版收边
4. 摘要缺失、正文过短、增强无差异等空态补齐
5. 小窗口适配继续压测
6. 模式记忆与复制 / 分享一致性优化
7. 图片相关体验后置补齐

## 最终希望达到的实现效果

这里用几种典型文章说明最终形态。

### 场景 1：长文类 RSS

用户点进一篇较长文章后：

- 默认先看到更适合阅读的模式
- 标题下方能快速看见来源、日期、阅读模式按钮
- 左侧有轻量悬浮目录
- 需要时可快速跳到关键部分
- 如果切回原始模式，用户能明显感觉到这是“原始解析版”

最终观感：

- 像一个轻量阅读器
- 而不是把整坨 Markdown 直接塞进详情页

### 场景 2：纯资讯 / 短讯 RSS

用户点进短内容后：

- 如果有摘要，默认直接进入摘要模式
- 如果增强和原始其实一样，会有清楚提示
- 不会出现过重的图集、折叠和复杂提示

最终观感：

- 短内容读得更快
- 长内容读得更稳

### 场景 3：异常与降级场景

当 RSS 源本身质量不高时：

- 没摘要，会说明原因
- 正文太短，会说明是源内容有限
- 图片失效，会给占位
- 外链仍可继续打开

最终观感：

- 即使源数据不完美，阅读体验也不会突然“坏掉”

## 到达“阅读器基本收尾”的验收标准

满足下面这些时，可以认为阅读器层基本收尾：

- 三种阅读模式切换稳定，含义清楚
- 悬浮目录稳定、轻盈、跳转顺畅
- 长文无需折叠也能快速定位和跳转
- 正文、附件、标签、元信息的层级统一
- 空态、异常态、无摘要态都有明确提示
- 默认小窗口和正常窗口下都保持整齐
- 不同类型 RSS 在同一套 UI 下看起来仍然有秩序

## 涉及代码区域

后续如果继续完善阅读器，主要会落在这些文件：

- `frontend/js/app.js`
- `frontend/index.html`
- `frontend/css/styles.css`

如果后续只是阅读层优化，原则上不需要新增后端模块。
