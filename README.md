# MicroFlow

MicroFlow 是一个面向校园信息场景的本地桌面应用，用来聚合校内通知、公文和自定义订阅源，并结合 AI 对内容进行提炼、重排和分享。

当前项目仍处于开发阶段，尚未正式发布。本文档用于持续记录当前进度、功能边界和开发约定，后续每次有较大更新时都应同步修改。

## 项目目标

MicroFlow 的核心目标是降低校园信息检索和阅读成本：

- 聚合多来源校园通知，而不是反复登录多个站点逐个查看。
- 对长文本通知进行 AI 提炼，突出时间、地点、对象、动作和关键要求。
- 允许用户只订阅自己关心的来源，减少信息过载。
- 支持将阅读结果复制、截图和分享。
- 在本地运行，尽量保持“本地数据、本地配置、本地处理”的使用体验。

## 当前已实现的能力

### 1. 多源信息聚合

- 内置多组校内信息源爬虫，包括公文通和多个学院站点。
- 支持统一调度、分页加载、来源筛选和按时间排序展示。
- 支持按订阅来源控制列表展示范围。
- 支持搜索、未读统计、收藏和软删除。

### 2. 自定义数据源

- 支持新增自定义 HTML 网页规则。
- 支持通过 AI 分析页面结构并生成 CSS 选择器规则。
- 支持规则保存、启停、删除和测试。
- 支持编辑已有 HTML 规则时直接重新测试样本，手动微调选择器后可立即验证。
- HTML 规则在修改选择器、目标 URL 或正文抓取策略后，现会强制要求重新测试，避免把未验证规则直接保存。
- 编辑已有 HTML 规则时，如提取字段变化较大，现可直接重新生成规则，并继续更新原有规则记录。
- 支持 RSS/Atom 订阅源，无需 CSS 选择器即可接入。
- 支持编辑 RSS 规则时在保存前重新验证订阅源可用性，避免误存无效地址。

### 3. RSS 阅读增强

- 支持 RSS/Atom feed 解析。
- 支持 RSS 内容标准化，提取正文、链接、图片、附件和结构化块内容。
- RSS 结构化块已细化到更丰富的元数据层，标题块会带锚点 ID，正文/引用/列表会保留更细的 inline segment 信息，便于后续阅读器继续打磨。
- 支持 RSS 正文裸链清洗：会尽量把 `http/https` 裸链整理成更清晰的 Markdown 链接形式，再进入 AI 与阅读区。
- 支持 RSS 详情页回源补全，处理“feed 只给摘要”的情况。
- 支持 RSS 源级策略首轮落地：预览与保存时会自动识别“资讯类 / 长文类 / 图文类”，并绑定对应模板默认值。
- 支持 RSS 模板库手动选择：规则表单可直接指定“快讯精读 / 长文提炼 / 图文摘要”等模板，也可保留自动推断。
- 支持 RSS AI 结果缓存复用：同一份正文在相同模型与提示词下，可直接复用已有排版/摘要结果，减少重复请求。
- 支持 RSS 长文自动分块摘要：面对超长正文时，先按段落分块提炼，再做最终汇总，降低超时和失败重试概率。
- 支持 RSS 的原始、增强、摘要三种阅读模式。
- 支持 RSS 规则预览中的结构化诊断提示，可在保存前识别正文缺失、正文偏短、图多文少和 AI 预览状态。
- 支持 RSS 规则健康状态首轮落地：规则列表会显示最近抓取状态、连续失败次数，并记录最近检查与错误信息。
- 支持 RSS 健康汇总面板：会汇总正常/空结果/异常数量，并列出需要注意的异常源。
- 支持 RSS 图片加载失败时的前端降级与占位提示。
- 支持 RSS 图片站内预览，减少频繁跳浏览器查看原图。
- 支持 RSS 长文默认折叠预览，并可手动展开/收起。
- 支持 RSS 重新生成时按当前规则同步 AI 结果，区分“仅排版 / 仅摘要 / 排版+摘要 / 清空旧 AI 结果”。
- 已兼容旧版 RSS `summary` 字段，历史测试数据仍可正确进入摘要模式、标签展示和摘要编辑链路。
- RSS 详情默认会根据是否存在独立摘要自动落到“摘要 / 增强”模式，后台静默刷新列表时也不会再误改当前阅读模式。
- 支持 RSS 专属 AI 配置：
  - AI 排版增强
  - AI 摘要与标签
  - 拆分提示词

### 4. AI 能力

- 支持 OpenAI 兼容 API 接口。
- 支持配置 `Base URL`、`API Key`、`Model Name`。
- 支持对文章执行 AI 总结、标签提取和 Markdown 重排。
- 支持基于正文内容和提示词的 AI 缓存键，避免预览、批量抓取和手动重生成时对同一内容重复调用模型。
- 支持手动重新生成总结。
- RSS 标签现已在返回层补充优先级元数据，保持首个标签为最高优先级，便于后续继续打磨摘要标签展示。
- 支持 AI 调用取消机制。
- 支持多模型配置，用于自定义规则生成时的辅助投票。

### 5. 本地数据与处理链路

- 使用本地 SQLite 存储文章、摘要、附件、收藏和状态信息。
- 数据库存储已包含：
  - 文章基础信息
  - 已读/收藏/删除状态
  - 数据源类型
  - 结构化内容块
  - 图片资产
  - AI 相关字段
- 后端已拆分为调度、抓取、文章处理、数据库写入、配置管理等模块。
- 数据写入采用队列化串行写，读操作采用连接池并发读取。

### 6. 桌面端体验

- 使用 `pywebview` 作为桌面应用壳层。
- 前端为本地静态页面，和 Python 后端通过 `js_api` 交互。
- 支持系统托盘、静默运行、未读红点和免打扰模式。
- 支持窗口置顶、隐藏到托盘、启动时最小化。
- 支持文章截图导出与剪贴板复制。

### 7. 其他能力

- 支持邮件推送配置与测试发送。
- 支持网络访问状态检查。
- 支持启动时安全检查与版本信息获取。
- 支持本地字体导入。
- 支持 PyInstaller 打包为桌面应用。

## 运行环境

### 当前推荐环境

- Python 3.10+
- macOS 优先
- Apple Silicon 环境已有明确打包配置

### 说明

- 当前项目主开发和打包配置明显偏向 macOS。
- macOS 下优先使用 `PyObjC` 原生托盘方案。
- 其他平台理论上可走 `pystray` 备选逻辑，但尚未作为当前主验证环境。
- 内置校内站点数据源很多依赖校园网环境；如果脱离校园网，内置源可能无法访问。
- RSS 和公网自定义源对校园网依赖较低。

## 快速开始

### 1. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动项目

```bash
python main.py
```

应用启动后会打开桌面窗口，前端页面由本地 `pywebview` 加载。

## 初次使用建议

首次启动后，建议优先完成以下配置：

1. 配置 AI 接口信息：
   - `API Base URL`
   - `API Key`
   - `Model Name`
2. 勾选需要订阅的内置来源。
3. 如需接入外部信息源，可新增：
   - 自定义 HTML 数据源
   - RSS 订阅源
4. 如需邮件提醒，可补充 SMTP 和订阅邮箱配置。
